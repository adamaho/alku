Users and companies often have a preference for the accent color of their applications. Alku supports this using the `data-accent-color` attribute. The accent color attribute modifies the `--ak-accent-color-<value>` css variables which will affect any components that use those variables.

```tsx title="root.tsx" ins="class="alku"" ins="data-accent-color="purple""
export default function Root() {
  return (
    <html>
      <head></head>
      <body class="alku" data-accent-color="purple">
        <App />
      </body>
    </html>
  );
}
```

Alku comes with seven colors that are available for use with `data-accent-color`.

## Branding

Alku allows you to configure an additional brand accent color that can be used to style your components and application using the `brand` accent color.

Luckily, the community has started to create some amazing tools for generating color palettes. Alku's color palette is based on the one defined by [Radix Colors](https://www.radix-ui.com/colors/custom). We suggest you start there when it comes to configuring your brand color.

Each color can be overridden using css variables defined in a global css file in your application. For example, if you want to define a custom brand color for your application you can do the following:

Once you have configured your custom brand color you can set `data-accent-color` to `brand` and all of your components will use that color as the accent.

```tsx title="root.tsx" ins="data-accent-color="brand""
export default function Root() {
  return (
    <html>
      <head></head>
      <body class="alku" data-accent-color="brand">
        <App />
      </body>
    </html>
  );
}
```
